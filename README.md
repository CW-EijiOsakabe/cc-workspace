# cc-workspace

## 概要

Claude Code向けの各種モジュールを提供するワークスペース。AIエージェントを活用したリサーチ、プロトタイピング、シミュレーションをそれぞれ独立したモジュールとして構造化し、スキル（`/コマンド`）から操作する。

### モジュール一覧

| モジュール | ディレクトリ | 用途 |
|-----------|-------------|------|
| Topic | `topics/` | AIリサーチ・ナレッジ管理 |
| Lab | `lab/` | 技術検証・MVP作成 |
| Sim | `sims/` | マルチエージェント変遷シミュレーション |

## セットアップ

```
/common-setup
```

`index.md` の作成など、ワークスペースの初期構築を行う。新しい環境で最初に1回実行する。

## Topic モジュール（調査）

トピック（調査テーマ）ごとにフォルダを作成し、Web調査・議論・結論までの一連の流れを構造化して記録する。過去のトピックはタグで分類され、`topics/index.md` から横断的に検索・参照できる。

### クイックスタート: `/topic-start` の使い方

#### モード一覧

| モード | 末尾キーワード | エージェント数 | 議論方式 | 向いているテーマ |
|--------|---------------|---------------|---------|----------------|
| 通常 | なし | 1 | なし | シンプルな調査・情報収集 |
| チーム並列 | `ぞす` | 2 | 各自が別ファイルに書く | 多角的な比較検討 |
| チームメッセージ | `ぞすめ` | 2 | メッセージ交換（最大10往復） | 対話的に深掘りしたいテーマ |
| Devil's Advocate | `ぞすでびる` | 3（提案者・反論者・仲裁者） | 3者間議論（最大20メッセージ） | 賛否が分かれるテーマの多面的検証 |

テーマ名の末尾キーワードで調査モードが変わる。

#### プロンプトサンプル
```sh
# 通常
/topic-start Claude Codeのhooks機能について調査

# チーム並列
/topic-start マイクロサービス vs モノリスの比較検討ぞす

# チームメッセージ
/topic-start Claude Code vs Cursor比較ぞすめ

# Devil's Advocate
/topic-start TypeScript導入の是非についてぞすでびる
```

### 基本的な使い方

```bash
# 新しい調査トピックを開始（上記4モードのいずれか）
/topic-start テーマ名

# 既存トピックに追加調査を実行（research.md / conclusion.md に追記）
/topic-research 追加の調査クエリ

# 過去のトピックをキーワード・タグで検索
/topic-lookup
```

## Lab モジュール（技術検証）

技術的な調査やMVP作成などのお題ごとに専用フォルダを作成し、ヒアリングを経て `steering.md`（方針書）を生成する。Topicモジュールが「調べる」ことに特化しているのに対し、Labは「作る・試す」ことに重点を置く。

### クイックスタート

```sh
# 新しいLabを開始（お題フォルダ作成→ヒアリング→steering.md作成）
/lab-start WebSocketでリアルタイムチャットを作る
```

### 基本的な使い方

```bash
# 新しいLabを開始
/lab-start お題

# 作業はLabフォルダ内で自由に進める
# lab/sessions/YYYYMMDD-<slug>/ 配下にコードや成果物を配置
```

### ディレクトリ構成

```
lab/
└── sessions/
    └── YYYYMMDD-<slug>/
        ├── steering.md       # 方針書（ゴール・前提・アプローチ）
        ├── src/              # ソースコード（お題による）
        └── ...               # その他成果物
```

## Sim モジュール（シミュレーション）

LLMエージェントが複数のステークホルダーを演じ、四半期ごとのターン制で相互作用しながら、あるテーマがどう変遷するかをシミュレートするフレームワーク。Web調査に基づいて初期設定を生成し、エージェント間の意思決定・イベント抽選・二次効果の連鎖を自動で進行する。

### クイックスタート

```sh
# 新規シミュレーション作成（お題→Web調査→設定提案→ファイル生成）
/sim-new AI時代の工数管理SaaS市場の変遷

# シミュレーション実行（4四半期分進める）
/sim-run 4
```

### 基本的な使い方

```bash
# 新規シミュレーション作成
/sim-new テーマ

# シミュレーション実行（デフォルト: 1四半期）
/sim-run [sim-path] [四半期数]

# 特定シナリオ/ブランチのみ実行
/sim-run --scenario baseline
/sim-run --branch platform-bundling
```

### シナリオモード

| モード | 説明 | 向いているケース |
|--------|------|----------------|
| 事前定義型（predefined） | Baseline / Bullish / Bearish などの固定シナリオで比較 | 明確な比較軸がある時 |
| 動的分岐型（dynamic） | 1本のシミュレーションを進め、途中で分岐点を探索 | 予想外の展開を許容したい時 |

### ディレクトリ構成

```
sims/
├── src/helpers/           # TSヘルパー（共通ユーティリティ）
├── sessions/
│   └── YYYYMMDD-<slug>/
│       ├── steering.md        # シミュレーション設定
│       ├── research-context.md # Web調査結果
│       ├── agents/            # エージェントペルソナ定義
│       ├── events/            # イベントプール
│       ├── world-state.json   # 世界状態（数値パラメータ）
│       ├── scenarios/         # 【事前定義型】シナリオ別データ
│       ├── branches/          # 【動的分岐型】分岐ブランチ
│       └── timeline/          # タイムライン記録
└── package.json
```
